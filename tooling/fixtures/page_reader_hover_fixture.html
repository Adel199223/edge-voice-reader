Hover Fixture

Use this page to validate that the page-reader chip appears beside readable blocks, stays visible while moving onto it, and starts reading from the intended paragraph.

Speechify-Style Hover Chip Fixture

This article exists only to validate the paragraph play affordance for Edge Voice Reader. The content column is intentionally narrow enough to expose chip placement problems while leaving a consistent gutter beside each readable block.

When the hover behavior is correct, the chip should appear quickly, stay alive while the pointer travels from the paragraph to the chip, and launch reading from the chosen block without flicker or accidental disappearance.

Expected Behavior

The chip should feel small and supportive rather than dominant. It should read like an inline control attached to the paragraph, not a floating bubble that lands on top of the content.

This blockquote is a useful secondary target because it has padding and a custom border, which makes gutter placement easier to spot visually.
  • The chip should prefer a stable left gutter when room exists.
  • On tighter layouts it may inset slightly, but it should still avoid covering the text body.
  • Clicking the chip should always read from the matching block.

Regression Notes

The previous implementation used a larger floating control positioned into the block area and tied activation to transient hover state. This page makes it easy to see whether the new pinned-target behavior fixes that race condition.

Suggested local serving command: python3 -m http.server 8765 from the repo root, then open http://127.0.0.1:8765/tooling/fixtures/page_reader_hover_fixture.html.
